# Java 異常處理基礎
在 Java 中，異常（Exception）是程式執行時發生的錯誤。Java 將異常分為以下幾類：

*   **Checked Exception（受檢例外）：** 這些異常在編譯時被檢查，並且**必須**顯式處理（例如，使用 `try-catch` 區塊）。如果方法可能會拋出受檢例外，則必須在方法簽名中使用 `throws` 關鍵字宣告。`IOException` 是一個常見的受檢例外，其他也有如`ArithmeticException`。
*   **Unchecked Exception（未受檢例外）：** 這些異常在編譯時不被檢查，並且**不需要**顯式處理。它們通常是由程式錯誤引起的，例如空指標異常（`NullPointerException`）或陣列索引超出範圍異常（`ArrayIndexOutOfBoundsException`）。
*   **Error（錯誤）：** 錯誤表示嚴重的問題，通常是程式無法恢復的。`OutOfMemoryError` 是一個常見的錯誤。通常不應嘗試捕捉錯誤。

*   **try：** `try` 區塊包含可能拋出異常的程式碼。
*   **catch：** `catch` 區塊捕獲特定類型的異常。可以有多個 `catch` 區塊來處理不同類型的異常。
*   **finally：** `finally` 區塊包含無論是否發生異常都會執行的程式碼。這通常用於釋放資源，例如關閉檔案或資料庫連線。

## 範例：Exception/ExceptionExample.java

[Exception/ExceptionExample.java](Exception/ExceptionExample.java) 
`try` 區塊包含除以零的程式碼，這會拋出 `ArithmeticException`。`catch` 區塊捕獲這個異常並印出錯誤訊息。程式在異常處理後繼續執行。

## 範例：IOStream/FileCopyExample.java

[IOStream/FileCopyExample.java](IOStream/FileCopyExample.java) 使用 `try-with-resources` 語法來自動關閉 `InputStream` 和 `OutputStream`，以確保資源在程式碼執行完畢後被釋放，也使用了 `try-catch` 區塊來捕捉 `IOException`，因此即使發生 `IOException` 也能正常關閉，並在發生錯誤時印出堆疊追蹤。
