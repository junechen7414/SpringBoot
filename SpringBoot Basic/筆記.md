# Spring Boot 基本

RESTful API、IoC (依賴注入)、AOP (切面導向程式設計) 與 Gradle 套件管理工具，均為 Spring Boot 開發中的基礎架構，通常會在專案中同時存在並互相配合。

- Restful API
    *   `demo` 專案範例：`HelloController.java` 是一個 RESTful API 的範例。
        *   `@RestController` 註解標示此類別為 REST 控制器。
        *   `@GetMapping("/api/hello")` 註解將 `/api/hello` 路徑映射到 `hello()` 方法，使用 GET 方法處理請求。
        *   `hello()` 方法返回一個字串 "Hello from Service"，作為 API 的回應。
- IoC (依賴注入)
    *   說明：IoC (Inversion of Control，控制反轉) ，用於降低元件之間的耦合度。Spring 透過依賴注入 (Dependency Injection, DI) 來實現 IoC。在 DI 中，元件的依賴關係由 IoC 容器負責管理，而不是由元件自身建立。
    *   `demo` 專案範例：`HelloController.java` 展示了依賴注入的應用。
        *   `@Autowired` 註解用於自動注入 `HelloService` 的實例到 `HelloController` 中。
        *   Spring IoC 容器負責建立 `HelloService` 的實例，並將其注入到 `HelloController` 中，`HelloController` 不需要自行建立 `HelloService` 的實例。
- AOP (切面導向程式設計)
    *   說明：AOP (Aspect-Oriented Programming，切面導向程式設計) 是一種程式設計範式，用於將橫切關注點（例如日誌、安全性等）從業務邏輯中分離出來，以提高程式碼的模組化和可維護性。
    *   `demo` 專案範例：`LoggingAspect.java` 是一個 AOP 的範例。
        *   `@Aspect` 註解標示此類別為一個切面。
        *   `@Component` 註解讓 Spring IoC 容器管理此切面元件。
        *   `@Before("execution(* com.example.demo.*.*(..))")` 註解定義了一個前置通知 (Before Advice)，指定在 `com.example.demo` 套件下的所有類別的所有方法執行前，執行 `logBefore()` 方法。
        *   `logBefore()` 方法使用 `JoinPoint` 介面獲取方法簽章，並輸出日誌訊息。
- Gradle 套件管理工具
    *   說明：Gradle 是一個自動化建置工具，用於管理專案的依賴、編譯程式碼、執行測試、打包應用程式等。Spring Boot 專案通常使用 Gradle 來管理依賴和建置專案。
    *   `demo` 專案範例：
        *   `build.gradle` 檔案定義了專案的依賴、外掛和建置設定。
        *   `dependencies` 區塊列出了專案的依賴，例如 `implementation 'org.springframework.boot:spring-boot-starter-web'` 依賴用於支援 Web 開發。
        *   可以使用 `./gradlew build` 命令來編譯和打包專案。
